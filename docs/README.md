# Cursor Dashboard Docs

This documentation system is the durable product memory for `Cursor Dashboard`.
It captures why the product exists, who it serves, what research should guide design,
what is implemented today, and how future agents should reason about next steps.

## Sections

- `north-star/`: mission, positioning, values, and product principles
- `personas/`: progression stages, context personas, and high-value user flows
- `research/`: gamification, Octalysis, motivation psychology, and developer motivation
- `product/`: current state, implemented features, roadmap, open questions, and feature opportunities
- `memory/`: handoff notes, decision log, glossary, and canonical status model

## How To Use This Docs Tree

- Start in `north-star/` to understand product intent and decision filters.
- Read `personas/` before proposing features or UX changes.
- Use `research/` to distinguish evidence-backed guidance from product judgment.
- Check `product/current-state.md` and `product/implemented-features.md` before describing existing behavior.
- Update `memory/decision-log.md` and `memory/agent-handoff.md` when major direction changes.

## Recommended Reading Order

1. `north-star/mission-and-values.md`
2. `north-star/product-principles.md`
3. `personas/progression-stages.md`
4. `research/design-principles-summary.md`
5. `product/current-state.md`
6. `memory/agent-handoff.md`

## Ground Rules

- Keep docs factual about current implementation.
- Mark future ideas with the status model instead of implying they already exist.
- Prefer focused files over catch-all notes.
