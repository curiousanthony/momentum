# Design Principles Summary

This file is the shortest reference for future agents making feature decisions.

## Always Do

- make invisible progress visible
- optimize for competence before novelty
- reward recovery, not just perfect streaks
- compare users against themselves over time
- connect metrics to real developer value
- keep the tone premium, credible, and lightly gameful

## Usually Do

- frame progression as a growth journey
- surface one or two clear next-step opportunities
- use unlocks to reveal richer insight, not artificial scarcity

## Avoid

- vanity engagement loops
- public leaderboards as a core mechanic
- childish badges without meaning
- over-prescriptive coaching
- heavy dependence on fear of loss

## Quick Heuristic

If a mechanic would still feel useful after removing points and badges, it is
probably aligned with the product.
