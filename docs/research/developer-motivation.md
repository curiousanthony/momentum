# Developer Motivation

## Evidence

Software developers are motivated by more than rewards. Research summaries and
industry syntheses repeatedly point to:

- mastery and learning
- autonomy in how work is done
- technical challenge
- ownership
- purpose and usefulness

This matters because `Cursor Dashboard` is aimed at serious engineers, founders,
and ambitious builders rather than a generic consumer audience.

## Interpretation

If the dashboard feels childish, coercive, or disconnected from real work, this
audience will reject it quickly. If it feels credible, insightful, and
empowering, the gameful layer can reinforce motivation without reducing trust.

## Product Implications

- prioritize capability growth over generic engagement
- frame progression as professional development, not novelty
- connect behavior to leverage, craft, and outcomes
- use premium tone and concise language
- treat the user's time and judgment with respect
