# Status Model

Use these statuses consistently in `docs/product/` and future planning docs.

- `implemented`: exists in the product today and is traceable to code
- `discussing`: under exploration or brainstorming
- `planned`: approved for implementation planning but not started
- `building`: actively being implemented
- `blocked`: cannot proceed until a dependency or decision is resolved
- `shipped`: completed and considered released in the intended workflow
- `deprecated`: intentionally no longer recommended or maintained

## Usage Rules

- Do not mark ideas as `implemented` just because they appear in docs.
- Use `discussing` for concept-stage opportunities and unresolved ideas.
- Use `planned` only after direction has been approved and implementation planning has begun.
